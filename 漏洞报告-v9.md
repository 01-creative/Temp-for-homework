## 一、本次重点修复 — XXE 外部实体注入

v9alpha 的 `/xml-import` 路由用正则提取 `<!ENTITY SYSTEM "路径">` 后直接 `open()` 读取任意文件，攻击者通过构造恶意 XML 可读取服务器上任意文件。

### 漏洞1：通过外部实体读取本地任意文件

#### 漏洞原理

v9alpha 用正则匹配 `<!ENTITY` + `SYSTEM` 声明，提取文件路径后直接 `open()` 读取，不校验路径。攻击者构造 `SYSTEM "file:///etc/passwd"` 即可将系统文件内容嵌入解析结果。

#### 漏洞位置

| 文件 | 行号 | 说明 |
|------|------|------|
| `v9alpha/app.py` | 1566 | 正则提取 ENTITY SYSTEM 文件路径 |
| `v9alpha/app.py` | 1582 | `open(filepath).read()` — 无路径白名单 |

#### 攻击方式

```xml
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<users>
  <user><name>&xxe;</name><email>test@test.com</email></user>
</users>
```

| Payload | 泄露内容 |
|---------|----------|
| `file:///etc/passwd` | 系统用户数据库 |
| `file:///etc/hostname` | 主机名 |
| `file:///path/to/app.py` | 完整应用源码 |

#### 修复方案（v9beta）

多层防线：
1. **关键字拦截**：检测 `<!DOCTYPE`、`<!ENTITY`、`SYSTEM`、`PUBLIC` (忽略大小写)，直接拒绝
2. **defusedxml**：安全解析器，外部实体/DTD/XInclude 全部禁用
3. **大小限制**：100KB 上限
4. **友好错误提示**：不泄露路径或异常详情

#### 效果验证

| Payload | v9alpha | v9beta |
|---------|:--:|:--:|
| 普通 XML（含 user 节点） | ✅ 正常 | ✅ 正常 |
| `<!DOCTYPE ... SYSTEM "file:///etc/passwd">` | 🔴 泄露 | ✅ 拦截 |
| `<!ENTITY` 声明 | 🔴 实体注入 | ✅ 拦截 |

## 三、本次新增：头像上传完善

### 问题

1. 上传成功后头像未存储到用户记录 → 首页和个人中心不显示
2. 未校验图片尺寸（宽高）
3. 文件大小限制未独立于总体 MAX_CONTENT_LENGTH

### 修复

| 修改 | 说明 |
|------|------|
| 数据库新增 `avatar TEXT` 列 | 存储用户头像文件名 |
| 上传成功后 `UPDATE users SET avatar = ?` | 关联用户与头像 |
| Pillow 读取尺寸，最大 2000×2000 | 防止超大分辨率图片 |
| 文件大小上限 5MB | 针对头像的合理限制 |
| `db_get_user_safe` 包含 avatar 字段 | 查询时返回头像文件名 |
| 首页 + 个人中心显示头像 | 120px/150px 圆形展示 |
| Polyglot 图片重编码（M1 修复） | Pillow 重新编码剥离 PHP 等附加数据 |
