#### C2 — HTTPS DNS 重绑定绕过 (SSRF)

| 项目 | 内容 |
|------|------|
| **发现代理** | 白盒#1 · 白盒#2 |
| **影响文件** | `app.py` 第 1459-1462 行 |
| **描述** | 对于 HTTPS URL，`/fetch-url` 端点丢弃已验证的 IP 地址，直接将原始 URL 传给 `urllib.request`。攻击者利用短 TTL 的 DNS 记录：首次解析返回合法公网 IP（通过安全检查），随后切换 DNS 指向 `127.0.0.1` 或云元数据 IP `169.254.169.254`，`urllib.request` 重新解析后连接到内部地址，响应内容返回给攻击者。 |

**原代码：**
```python
else:
    # HTTPS：保持原有 URL（无法用 IP 替换，否则 TLS 证书校验会失败）
    actual_url = url
    extra_headers = {}
```

**修复方案：**
```python
else:
    # HTTPS with DNS rebinding protection:
    # Re-verify DNS hasn't changed right before connecting
    import ssl as _ssl
    try:
        recheck_ips, _ = _resolve_and_check_ips(hostname)
        if recheck_ips is None:
            return render_template("index.html", ...)
        if set(safe_ips) != set(recheck_ips):
            return render_template("index.html",
                url_error="DNS 解析结果已变更，为安全起见已拒绝请求", ...)
    except Exception:
        return render_template("index.html", ...)
    actual_url = url
    extra_headers = {}
```

**修复状态:** ✅ 已修复 — 连接前执行 DNS 二次校验，结果不一致则拒绝请求

#### C3 — NaN 余额充值业务逻辑绕过

| 项目 | 内容 |
|------|------|
| **发现代理** | 黑盒#1 |
| **影响文件** | `app.py` 第 1085-1098 行 |
| **描述** | `float("NaN")` 通过了所有数值验证：`NaN <= 0` 返回 `False`、`NaN > 100000` 也返回 `False`。NaN 值绕过验证后执行 SQL `balance + NaN`，将用户余额损坏为 NULL。 |

**利用方式：**
```bash
curl -X POST http://127.0.0.1:5050/recharge \
  -d "amount=NaN&csrf_token=<token>"
# 结果：余额从 99999 变为 NULL
```

**原代码：**
```python
amount = float(amount_str)
if amount <= 0:          # NaN <= 0 → False (绕过!)
    return error
if amount > 100000:       # NaN > 100000 → False (绕过!)
    return error
```

**修复方案：**
```python
import math

amount = float(amount_str)
if math.isnan(amount) or math.isinf(amount):
    return render_template(..., recharge_error="金额必须是有效的数字")
```

**修复状态:** ✅ 已修复 — NaN/Infinity 值被拒绝，返回错误提示"金额必须是有效的数字"

#### H5 — 登出后会话可重放

| 项目 | 内容 |
|------|------|
| **发现代理** | 白盒#2 · 黑盒#1 · 黑盒#2 |
| **影响文件** | `app.py` 第 899-906 行 |

**修复方案：** 登出时递增 `session_version` 使旧会话失效
```python
@app.route("/logout", methods=["POST"])
def logout():
    ...
    if uid:
        conn = get_db()
        try:
            conn.execute(
                "UPDATE users SET session_version = session_version + 1 WHERE uid = ?",
                (uid,))
            conn.commit()
        finally:
            conn.close()
    session.clear()
    return redirect(url_for("index"))
```

**修复状态:** ✅ 已修复 — 登出后使用相同 Cookie 访问 /profile 返回 302 重定向
